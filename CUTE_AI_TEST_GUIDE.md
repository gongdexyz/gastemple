# 软萌 Pepe 佛祖 AI 测试指南 🐸✨

## 快速测试步骤

### 1. 启动开发服务器
```bash
npm run dev
```

### 2. 测试抽签功能

#### 访问抽签页面
- 导航到 **Gacha Page**（抽签页面）
- 点击 **"Draw Fortune"** 按钮

#### 检查佛祖判词
- ✅ 查看顶部的佛祖判词是否使用软萌文案
- ✅ 检查是否有可爱的 emoji（💕😊🌸等）
- ✅ 确认语气是否温柔（"宝子"、"哇哦"、"嘻嘻"）

**示例文案**：
```
"哇哦~宝子今天运气不错呢！除了你买的那几个币...嘻嘻，开玩笑啦（才不是）💕"
```

### 3. 测试毒舌报告弹窗

#### 打开毒舌报告
- 抽签后会显示一个币种
- 点击查看详情，打开毒舌报告弹窗

#### 检查视觉样式
- ✅ **标题**：应该显示"佛祖的温柔提醒"（中文）或 "BUDDHA'S GENTLE REMINDER"（英文）
- ✅ **图标**：应该是爱心 💕，不是骷髅头 💀
- ✅ **圆角**：应该是超大圆角（像团子一样圆润）
- ✅ **颜色**：应该是粉色主题，不是红色警告
- ✅ **装饰**：应该有闪烁的星星 ✨
- ✅ **列表项**：前缀应该是粉色爱心 💕

#### 检查文案内容
- ✅ 毒舌分析的文案应该是软萌风格
- ✅ 应该有大量 emoji
- ✅ 语气应该温柔但犀利

**示例文案**：
```
"这走势比佛祖的心跳还平呢~ 庄家是去度假了，还是忘记你了呀？😴"
```

### 4. 测试中英文切换

#### 切换语言
- 点击右上角的语言切换按钮
- 切换到英文（EN）

#### 检查英文文案
- ✅ 佛祖判词应该显示英文软萌版
- ✅ 毒舌报告标题应该是 "BUDDHA'S GENTLE REMINDER"
- ✅ 文案应该保持可爱风格

**示例英文文案**：
```
"Wow~ Sweetie's luck is pretty good today! Except for those coins you bought... Just kidding! (Not really) 💕"
```

### 5. 测试不同场景

#### 场景 A：横盘币种（变化 < 5%）
- 应该显示哄孩子版文案
- 例如："这币已经睡着了哦~ 要不要戳一下看看还活着吗？💤"

#### 场景 B：暴跌币种（变化 < -10%）
- 应该显示温柔安慰版文案
- 例如："哎呀跌了好多呢~ 没事，跌了90%还有90%可以跌，空间很大哦~ 💫"

#### 场景 C：暴涨币种（变化 > 15%）
- 应该显示温柔提醒版文案
- 例如："哇塞涨了好多！这就拿不住了吗？注定是送外卖的命呢~ 🎉"

#### 场景 D：小涨币种（变化 5%-15%）
- 应该显示鼓励版文案
- 例如："小涨不错呢！佛祖保佑，继续拿稳~ 加油哦！💪"

## 常见问题排查

### 问题 1：文案没有变化
**可能原因**：
- 缓存问题
- 文件没有保存

**解决方案**：
```bash
# 清除缓存并重启
npm run dev
# 或者硬刷新浏览器 Ctrl+Shift+R
```

### 问题 2：样式没有变化
**可能原因**：
- Tailwind CSS 没有重新编译
- 类名写错

**解决方案**：
```bash
# 重启开发服务器
npm run dev
```

### 问题 3：图标没有显示
**可能原因**：
- lucide-react 版本问题
- 图标名称错误

**解决方案**：
```bash
# 检查 lucide-react 是否安装
npm list lucide-react
```

### 问题 4：emoji 显示异常
**可能原因**：
- 字体不支持 emoji
- 浏览器版本过旧

**解决方案**：
- 使用现代浏览器（Chrome、Firefox、Safari）
- 更新浏览器到最新版本
